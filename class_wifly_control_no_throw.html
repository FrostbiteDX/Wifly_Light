Detailed Description

Class to communicate with a Wifly_Light Hardware.

The WiflyControlNoThrow class allows the user to control the Wifly_Light hardware. This is a wrapper class to WiflyControl to catch all exceptions from the lower software layers and convert them into error codes, which is required for example for jni or iOS clients There are three target's at the Wifly_Light Hardware.

  • Bootloader
    All methodes with Bl* relate to the bootloader part.
  • Firmware
    All methodes with Fw* relate to the firmware part.
  • RN-171 Wifi Interface
    All methodes witch Conf* relate to the communication module.
